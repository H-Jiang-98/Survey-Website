# Pi05

## 一句话结论

[[Pi05]] 的核心不是发明一个全新 policy head，而是证明 [[Vision-Language-Action Model|VLA]] 通过大规模异构 [[Co-Training|共训练]] 可以获得真实新家庭环境中的 [[Open-World Generalization|开放世界泛化]]：它把移动机器人数据、其他机器人数据、Web 多模态数据、高层子任务标注和 verbal instruction 统一放进一个 VLA 训练流程里。

## 论文定位

- 任务：真实家庭中的 [[Mobile Manipulation|移动操作]]，包括厨房/卧室清理、放盘子、收纳物品、整理床铺、放衣物等。
- 方法：基于 $\pi_0$，加入异构数据共训练、[[High-Level Subtask Prediction|高层子任务预测]]、[[FAST Action Tokenizer|FAST 动作 token]] 和 [[Flow Matching|flow matching]] action expert。
- 机器人：两类 mobile manipulator，双 6 DoF 机械臂、平行夹爪、移动底盘、torso lift、4 个 RGB 相机。
- 关键问题：VLA 能不能从训练环境之外泛化到完全没见过的真实家庭，而不是只在训练相近的实验室 setup 中表现好。
- 主要结论：跨 embodiment 数据、Web 语义数据、高层子任务预测和 verbal instruction 都有可测收益；仅扩大目标机器人数据还不够。

## 核心贡献

1. **开放世界家庭泛化实验**：在没见过的真实家庭和 mock homes 中评估长程家务任务，任务持续数分钟，不只是单步抓取。
2. **异构训练配方**：把 MM、ME、CE、HL、WD、VI 六类数据组合到一个统一 VLA 训练流程里。
3. **高低层统一模型**：同一个模型先输出文本形式的高层子任务，再基于该子任务输出低层连续动作。
4. **离散与连续动作结合**：pre-training 阶段用 FAST token 做高效自回归训练，post-training 阶段加入连续 [[Action Expert|动作专家]] 支持实时控制。
5. **系统性 ablation**：分别量化 Web data、跨 embodiment 数据、高层推理和 verbal instruction 的作用。

## 背景问题

真实家务机器人的泛化不只是“见过更多物体”。一个厨房清理任务里同时包含多层泛化：

1. 低层物理泛化：拿起没见过的碗、盘子、餐具、衣物。
2. 场景泛化：新房间布局、新柜门、新抽屉、新桌面高度。
3. 语义泛化：知道什么该放进水槽、什么该放进抽屉、什么工具适合擦 spill。
4. 长程结构泛化：把“clean the bedroom”拆成若干可执行子任务。
5. 跨机器人迁移：利用静态臂、实验室机器人和开源数据提升目标 mobile manipulator。

传统 VLA 的问题是评估环境往往贴近训练环境。[[Pi05]] 直接把 evaluation 设在 unseen homes，因此它测的是 recipe 是否真的支撑 [[Open-World Generalization|open-world generalization]]。

## 方法详解

### 总体架构

[[Pi05]] 可以理解成在 $\pi_0$ 基础上做了两件事：一是把多种异构数据统一进 [[Vision-Language-Action Model|VLA]] 的 sequence modeling 框架，二是把执行时策略拆成“高层语义子任务 + 低层连续动作块”。它不是 planner + controller 的传统分层系统，而是同一个模型同时承担 [[High-Level Subtask Prediction|子任务预测]] 和 [[Action Chunking|动作块生成]]。

模型分布写成：

- 高层：$\pi_\theta(\hat{\ell}\mid \mathbf{o}_t,\ell)$，根据当前观测和总任务 prompt 输出下一步语义子任务。
- 低层：$\pi_\theta(\mathbf{a}_{t:t+H}\mid \mathbf{o}_t,\hat{\ell})$，根据当前观测和子任务生成 action chunk。

注意论文明确把低层动作分布条件化在 $\hat{\ell}$ 上，而不是直接条件化在原始长程 prompt $\ell$ 上。这样做的实际含义是：`clean the kitchen` 这类长程目标先被压缩成当前可执行的短语义目标，例如 `pick up the plate`，低层动作专家只需要解决这个局部操作。

### 输入与输出接口

[[Pi05]] 的输入是一个多模态 prefix：

- 输入：相机图像 $\mathbf{I}^1_t,\ldots,\mathbf{I}^n_t$、机器人状态 $\mathbf{q}_t$、语言任务 $\ell$。
- 图像来自多个 RGB 相机；真实机器人有 forward、backward、left wrist、right wrist 四个相机。
- 状态 $\mathbf{q}_t$ 包括关节角、夹爪 pose、torso lift pose 和 base velocity；附录里说明 proprioceptive state 被离散化后作为 text tokens 输入。
- 高层输出是 tokenized text，可以是子任务 $\hat{\ell}$，也可以在 web data / VQA task 中作为普通文本回答。
- 低层输出是 50-step 左右的连续 action chunk；附录写 action horizon 为 50，即 $H=49$。

低层 action chunk 控制的量包括：

- 双臂 joints / end-effector targets。
- 两个 parallel jaw grippers。
- base velocity，其中 base 是 2D linear velocity + 1D angular velocity。
- torso lift，随平台不同为 1D 或 2D。

移动机器人平台的 state/action space 总维度是 18 或 19。所有 action data 都被归一化到 $[-1,1]$，归一化范围来自各数据集每个 action dimension 的 1% 和 99% quantile；不同机器人 action 维度不一致时，低维机器人用 zero padding 对齐到统一最大维度。为了区分 joint target 和 end-effector target，prompt 中会加入类似 `<<control_mode>> joint/end effector <<control_mode>>` 的控制模式 token。

### 层级推理流程

运行时是一个两步 inference loop：

1. **高层推理**：给定总任务 prompt，例如 `clean the bedroom`，模型读取当前观测，生成文本子任务 $\hat{\ell}$，例如 `pick up pillow` 或 `adjust the blanket`。
2. **低层推理**：把 $\hat{\ell}$ 当成当前控制目标，并结合当前观测 $\mathbf{o}_t$，交给 [[Action Expert|action expert]] 生成连续动作块 $\mathbf{a}_{t:t+H}$。
3. **动作执行**：机器人以 50 Hz 执行 action chunk，目标位姿/速度由 [[PD Controller]] 跟踪。
4. **闭环更新**：执行若干步后重新观察，再重复高层/低层推理。

这个设计的价值是把长程 household task 的组合结构显式暴露给模型。高层子任务更像“语义当前意图”，低层 action expert 更像“针对这个意图的 visuomotor controller”。论文里的 examples 包括 `close the cabinets`、`put the items in the drawer`、`wipe the spill`、`put the dishes in the sink` 等高层任务，每个任务再分解为 pickup、open、place、close、wipe 这类短子任务。

这里容易误解的一点是：低层 action expert **不是只吃 $\hat{\ell}$ 这几个文本 token**。论文的分解是

$$
\pi_\theta(\mathbf{a}_{t:t+H},\hat{\ell}\mid \mathbf{o}_t,\ell)
=
\pi_\theta(\mathbf{a}_{t:t+H}\mid \mathbf{o}_t,\hat{\ell})\,
\pi_\theta(\hat{\ell}\mid \mathbf{o}_t,\ell)
$$

也就是说，原始长指令 $\ell$ 先通过高层推理压缩成当前子任务 $\hat{\ell}$；低层控制阶段概念上条件于当前观测 $\mathbf{o}_t$ 和这个子任务 $\hat{\ell}$，而不是只看 $\hat{\ell}$，也不是继续直接依赖原始长指令 $\ell$。实现上，action expert 读的是 VLM 对图像、状态和子任务文本形成的 prefix hidden states，而不是文本 logits 本身。

### Transformer 表示

模型本体仍是 transformer。论文用“token”泛指离散和连续输入：

- text token：语言 prompt、子任务文本、VQA/caption 的答案 token。
- image patch：来自各个相机的视觉 patch。
- action token：既包括 [[FAST Action Tokenizer|FAST]] 离散动作 token，也包括 [[Flow Matching|flow matching]] 中的 noisy continuous action token。

不同 token type 通过 $\rho(x_i)$ 选择不同 encoder / expert weights：

- image patch 先过视觉编码器。
- text token 用 embedding matrix。
- continuous action token 先线性投影到 transformer embedding space。
- action output 由单独的 [[Action Expert|action expert]] 负责。

输出也分两路：

- $y^\ell_{1:M}$：text logits，用于生成子任务、回答 VQA、预测 FAST action tokens 等。
- $y^a_{1:H}$：action expert 输出，经 final linear projection 解码成 flow vector field。

### VLM backbone 与 action expert

附录 A-E 给出的技术细节更接近实现：

- VLM backbone 使用 [[PaliGemma]] 初始化，约 2B 参数。
- action expert 是较小 transformer，约 300M 参数。
- 2B VLM 的配置沿用 $\pi_0$：width 2048、depth 18、mlp_dim 16384、num_heads 18、num_kv_heads 1、head_dim 256。
- action expert 配置类似，但 width 1024、mlp_dim 4096。
- action expert 输入长度是 50-step noisy action chunk，动作维度为统一 padded action dimension。

和 $\pi_0$ 不同的一点是 timestep 注入方式。$\pi_0$ 把 flow matching timestep $\tau$ 和 noisy action 先融合再送入 transformer；[[Pi05]] 用单独的 MLP 编码 $\tau$，再通过 adaptive RMSNorm 注入 action expert 的每一层。timestep MLP 的形式就是后面公式 5：

$$
\mathrm{swish}(W_2\cdot \mathrm{swish}(W_1\cdot \phi(\tau)))
$$

其中 $\phi$ 是 sinusoidal positional encoding。这个设计让 action token 本身和 denoising timestep 的条件信息解耦，结构上更清晰。换句话说，$\tau$ 是逐层注入的全局 denoising 条件，不是 action 序列中的一个额外 token。

### 离散动作与连续动作的双表示

论文的核心工程折中是：训练时想要 [[FAST Action Tokenizer|FAST]] 的效率，推理时想要 [[Flow Matching]] 的连续动作质量和非自回归速度。

两种表示的作用不同：

- **FAST 离散动作 token**：把 action chunk 压缩成 token，适合在大规模 pre-training 中用 next-token prediction 训练，吞吐高、实现简单。
- **连续 flow action expert**：post-training 阶段加入，用少量 denoising steps 直接生成连续动作块，更适合 50 Hz 真实控制。

因此 [[Pi05]] 训练目标里同时保留 text/FAST token cross entropy 和 continuous action flow matching。pre-training 阶段令 $\alpha=0$，相当于只做 token prediction；post-training 阶段设 $\alpha=10.0$，联合优化文本 token 和 action expert。

推理时的顺序是：先用标准自回归 decoding 生成子任务文本 $\hat{\ell}$，再用 10 个 denoising steps 生成连续 action chunk。

### Action expert 的结构、训练与推理

Action expert 可以更具体地看成一个 **条件 flow-matching action generator**。它不负责理解图像和语言的全部语义，而是接收 VLM / prefix 提供的上下文，并在该上下文下把 noisy action chunk 逐步变成可执行的连续 action chunk。

注意这里有两个不同的“动作路径”：pre-training 里的动作路径是 [[FAST Action Tokenizer|FAST]] 离散 action token，走大 VLM 的 causal next-token prediction；post-training 里才加入 continuous action expert。也就是说，pre-training 阶段并不存在一个已经以 causal attention 形式训练好的 continuous action expert。

**网络结构**：

1. 在低层控制调用中，VLM backbone 处理当前图像、proprioceptive state 和生成的高层子任务 $\hat{\ell}$，得到可被 action expert attend 的 prefix hidden states。
2. Action expert 不是读取 $\hat{\ell}$ 的文本 logits，而是通过 attention 读取上述 prefix 表示；这些 prefix 里同时包含场景视觉、机器人状态和当前语义目标。
3. Action expert 自己接收长度为 50 的 noisy action chunk $\mathbf{a}^{\tau,\omega}_{t:t+H}$，其中 $H=49$。
4. 每个 noisy action token 先经过 linear projection 映射到 action expert 的 embedding space。
5. Flow timestep $\tau$ 不作为普通 token，也不直接 concat 到 noisy action；它单独经过 timestep MLP，再通过 adaptive RMSNorm 注入 action expert 每一层。
6. Action expert 输出 $y^a_{1:H}$，再经过 final linear projection 得到 flow vector field。

换句话说，action expert 不是直接“一次回归动作”，而是在 flow matching 框架下预测从 noisy action 指向真实 action 的速度场。它的输出不是最终动作本身，而是用于 denoising / integration 的 vector field。

**Pre-train 与 post-train 中的差异**：

| 阶段            | Action 表示                                  | Action expert                               | 训练目标                                        | 目的                                            |
| ------------- | ------------------------------------------ | ------------------------------------------- | ------------------------------------------- | --------------------------------------------- |
| Pre-training  | FAST 离散 action token                       | 没有 continuous action expert；causal attention 只属于 FAST action-token 路径，等价于 $\alpha=0$ | next-token prediction                       | 高吞吐地学习机器人动作、语言、视觉和跨 embodiment 表示             |
| Post-training | FAST token + continuous noisy action chunk | 加入随机初始化的 300M action expert                 | token CE + flow matching loss，$\alpha=10.0$ | 专门化到真实家庭 mobile manipulation，并获得可实时执行的连续动作生成器 |

这种设计避免了两个极端：如果全程只用 FAST，推理时自回归动作解码太慢；如果从一开始就训练 continuous flow head，大规模异构 pre-training 的吞吐和稳定性会更差。Pi05 先用 FAST 把大规模离散序列训练跑稳，再在 post-training 中把连续 action expert 接上。

因此，把它理解成“pre-trained 大 Transformer 后面加一个 action expert”大体是对的，但要补两点：第一，action expert 更像一个带独立权重的小 Transformer，通过 attention 读 VLM prefix，而不是简单接在最后的 MLP head；第二，post-training 不是只让 expert 看 $\hat{\ell}$，而是让它在 $\mathbf{o}_t+\hat{\ell}$ 的上下文里对 noisy action 做 flow matching。

**训练目标**：

Action expert 的监督来自 flow matching。给定真实动作块 $\mathbf{a}_{t:t+H}$ 和噪声 $\omega$，先构造中间 noisy action：

$$
\mathbf{a}^{\tau,\omega}_{t:t+H}
=
\tau\mathbf{a}_{t:t+H}
+
(1-\tau)\omega
$$

模型学习预测目标 vector field：

$$
\omega-\mathbf{a}_{t:t+H}
$$

完整 post-training objective 是后面公式 4：文本/FAST token 走 cross entropy，continuous action expert 走 flow matching MSE。直观地说，cross entropy 保留“会说、会识别、会预测子任务”的能力，flow matching loss 让 action expert 学会把一段 noisy action trajectory 修正成真实轨迹。

**推理计算**：

1. 模型先自回归生成高层子任务 $\hat{\ell}$，例如从 `clean the bedroom` 得到 `pick up pillow`。
2. 初始化一个高斯噪声 action chunk $\omega\sim\mathcal{N}(0,I)$。
3. 以当前观测 $\mathbf{o}_t$ 和子任务 $\hat{\ell}$ 形成的 VLM prefix 为条件，把 noisy action chunk 输入 action expert。
4. Action expert 预测当前 flow timestep 的 vector field。
5. 用数值积分 / denoising step 更新 action chunk；论文推理时使用 10 个 denoising steps。
6. 得到连续动作块 $\mathbf{a}_{t:t+H}$ 后，以 50 Hz 发送给机器人底层 [[PD Controller]] 跟踪。

因此，推理阶段真正执行的是“高层文本生成 + 低层连续 flow 采样”的组合：高层仍然是自回归 token decoding，低层不再用 FAST 自回归动作 token，而是用 action expert 做少步连续 denoising。

### Attention mask 与信息流

Pi05 同时训练 FAST 动作 token 和 continuous action expert，所以必须防止两种动作表示互相“偷答案”。附录的 attention mask 规则是：

- image、prompt tokens、proprioceptive state 构成 full prefix，彼此可见。
- FAST action tokens 可以 attend prefix，也可以自回归地 attend 之前的 FAST action tokens。
- action expert embeddings 可以 attend prefix 和其他 action expert embeddings。
- action expert embeddings 不 attend FAST action tokens，避免连续动作 head 从离散动作 token 泄漏标签。
- VLM embedding 不 attend action expert；整体信息流是从 VLM/prefix 单向流向 action expert。

这解释了 Figure 18 的意义：它不是普通 causal mask，而是为了让“文本/FAST 任务”和“连续动作任务”在同一个 transformer 训练过程中共享语义 prefix，同时隔离两种 action supervision。

所以更准确的心智模型是：大 VLM 先把“当前看到了什么、机器人状态是什么、现在该做哪个子任务”编码进 prefix；post-training 新增的 action expert 在这个 prefix 条件下，用 mutual-visible 的 noisy action tokens 预测 flow vector field。$\hat{\ell}$ 决定当前语义目标，但视觉和 proprioceptive prefix 决定这个目标在当前场景里具体该怎么落到连续控制上。

### 两阶段训练

#### Stage 1: Pre-training

第一阶段训练 280k gradient steps。目标是把 web VLM 适配成能处理机器人数据的 generalist VLA。此时所有任务都表示成离散 token：文本、框、问答和机器人动作都进入同一个 next-token prediction 框架，动作由 [[FAST Action Tokenizer|FAST]] 编码。

这个阶段的重点不是让目标 mobile manipulator 已经最优，而是通过 [[Co-Training|co-training]] 建立跨场景、跨 embodiment、跨任务的表示。论文强调：第一阶段 97.6% 的 training examples 并不是目标 mobile manipulator household task，而是其他机器人或 web 等异构来源。

数据源：

| 缩写 | 数据源 | 作用 |
|------|--------|------|
| MM | mobile manipulator household data | 约 400 小时、约 100 个家庭环境，是最贴近测试任务的数据 |
| ME | multi-environment non-mobile robot data | 静态/非移动单臂或双臂机器人，更轻便，能在更多真实家庭采集，主要补 scene diversity |
| CE | cross-embodiment laboratory data | 实验室多机器人、多任务数据和 OXE，补 [[Cross-Embodiment Transfer|embodiment diversity]] 与技能覆盖 |
| HL | high-level subtask prediction | 给长程机器人数据人工标注语义子任务和相关 bounding boxes，让模型学会“下一步做什么” |
| WD | Web multimodal data | captioning、VQA、object localization，补语义、视觉常识和 household object grounding |

HL 数据值得单独强调。对 MM、ME、CE 中涉及多个子任务的 episode，作者人工标注每段对应的语义子任务，例如 `pick up the cutting board`、`rearrange the pillow`。模型训练时既要预测这些文本子任务，也要基于子任务预测动作，因此同一个模型自然具备高层 policy 和低层 policy 两种用法。

#### Stage 2: Post-training

第二阶段训练 80k steps。目标从“广泛适配机器人/视觉语言数据”切换到“真实家庭 [[Mobile Manipulation|mobile manipulation]] 上机可用”。这一阶段加入随机初始化的 continuous [[Action Expert|action expert]]，并用后面公式 4 的联合目标训练：

- text / FAST token prediction：保持高层子任务、语言理解和离散动作能力。
- flow matching action prediction：训练连续动作 expert，支持更细粒度、更实时的低层控制。

Post-training 数据包括：

- MM 和 ME 中成功且长度合适的 action episodes。
- WD，用来保持语义和视觉能力。
- HL 中对应 multi-environment 数据的部分。
- VI：专家用自然语言逐步指挥机器人完成任务，作为高层子任务输出的 demonstration。

Post-training 不再使用 CE 作为主要数据源，原因是这一阶段要更聚焦目标 robot family 和真实家庭分布，而不是继续最大化 embodiment diversity。

这里的 [[Verbal Instruction Demonstration|verbal instruction demonstration]] 很关键。VI 的采集方式不是传统遥操作关节，而是人类专家“用语言遥操作”已经学到的低层 policy：专家实时选择合适的子任务命令，让机器人逐步完成清理、收纳、整理等任务。这样得到的是高层子任务示范，直接监督 $\pi_\theta(\hat{\ell}\mid \mathbf{o}_t,\ell)$。它只占 high-level mobile manipulation examples 的约 11%，但 ablation 里去掉它会显著变差。

### 机器人控制接口

两类 mobile manipulator 都有：

- 两个 6 DoF arms。
- parallel jaw grippers。
- wrist-mounted RGB cameras。
- forward 和 backward facing cameras。
- wheeled holonomic base。
- torso lift mechanism。

高层推理使用全部四个相机，因为它需要理解全局场景和任务进度；低层推理使用 wrist cameras 和 forward camera，因为动作控制更依赖手眼局部观测和前方环境。

控制系统刻意保持简单：模型直接 command arm / gripper / torso lift 的 target poses，以及 base target velocity；这些 target 用简单 [[PD Controller]] 跟踪。没有额外 motion planning、trajectory optimization 或 collision checking。论文用这个设置证明：泛化能力主要来自 VLA 训练配方和数据，而不是手写规划模块。

## 关键公式

### 公式1: [[High-Level Subtask Prediction|高低层联合分解]]

$$
\pi_{\theta}(\mathbf{a}_{t:t+H},\hat{\ell}\mid\mathbf{o}_{t},\ell)
=
\pi_{\theta}(\mathbf{a}_{t:t+H}\mid\mathbf{o}_{t},\hat{\ell})
\pi_{\theta}(\hat{\ell}\mid\mathbf{o}_{t},\ell)
$$

**含义**：同一个模型先预测高层子任务 $\hat{\ell}$，再基于该子任务预测低层动作块。

**符号说明**：

- $\mathbf{o}_t=[\mathbf{I}^{1}_{t},\ldots,\mathbf{I}^{n}_{t},\mathbf{q}_{t}]$：多相机图像和机器人状态。
- $\ell$：原始高层任务 prompt，例如 `put away the dishes`。
- $\hat{\ell}$：模型生成的高层子任务，例如 `pick up the cup`。
- $\mathbf{a}_{t:t+H}$：动作块。

### 公式2: [[Action Chunking|观测与动作块接口]]

$$
\mathbf{o}_{t}
=
[\mathbf{I}^{1}_{t},\ldots,\mathbf{I}^{n}_{t},\mathbf{q}_{t}]
$$

$$
\mathbf{a}_{t:t+H}
=
[a_t,a_{t+1},\ldots,a_{t+H}]
$$

**含义**：模型不是只看单张图，也不是只输出单步动作，而是基于多视角观测和状态输出一段动作。

**符号说明**：

- $\mathbf{I}^{i}_{t}$：第 $i$ 个相机图像。
- $\mathbf{q}_{t}$：关节、夹爪、torso、base 等 proprioceptive state。
- $H$：action horizon；附录技术细节中 action expert 使用 50-step horizon，即 $H=49$。

### 公式3: [[Flow Matching|flow matching 插值]]

$$
\mathbf{a}_{t:t+H}^{\tau,\omega}
=
\tau \mathbf{a}_{t:t+H}
+(1-\tau)\omega,
\qquad
\omega\sim\mathcal{N}(0,\mathbf{I})
$$

**含义**：训练 action expert 时，将真实动作块和噪声动作块插值得到中间状态，让模型预测从噪声到真实动作的向量场。

**符号说明**：

- $\tau\in[0,1]$：flow matching 时间。
- $\omega$：高斯噪声动作。
- $\mathbf{a}_{t:t+H}^{\tau,\omega}$：带噪中间动作块。

### 公式4: [[FAST Action Tokenizer|离散 token]] + [[Flow Matching|连续动作]] 联合损失

$$
\mathbb{E}_{\mathcal{D},\tau,\omega}
\left[
H\left(x_{1:M},f^{\ell}_{\theta}(\mathbf{o}_{t},\ell)\right)
+
\alpha
\left\|
\omega-\mathbf{a}_{t:t+H}
-f^{a}_{\theta}
\left(
\mathbf{a}^{\tau,\omega}_{t:t+H},
\mathbf{o}_{t},
\ell
\right)
\right\|^{2}
\right]
$$

**含义**：第一项是文本/FAST token 的 cross entropy，第二项是 action expert 的 flow matching 损失。

**符号说明**：

- $H(\cdot,\cdot)$：cross entropy loss。
- $f^\ell_\theta$：文本 token 输出 head。
- $f^a_\theta$：动作专家输出 head。
- $\alpha$：离散 token loss 和连续 action loss 的权重；post-training 中设置为 10.0。

### 公式5: [[Action Expert|时间步注入 MLP]]

$$
\mathrm{swish}
\left(
W_2\cdot
\mathrm{swish}
\left(
W_1\cdot\phi(\tau)
\right)
\right)
$$

**含义**：附录中 action expert 用单独 MLP 编码 flow matching timestep，再通过 adaptive RMSNorm 注入每层动作专家。

**符号说明**：

- $\phi:\mathbb{R}\rightarrow\mathbb{R}^{w}$：sinusoidal positional encoding。
- $W_1,W_2\in\mathbb{R}^{w\times w}$：timestep MLP 参数。
- $\tau$：flow matching timestep。

## 实验

### Evaluation tasks

| 任务 | 场景 | 计分方式 |
|------|------|----------|
| Dishes in Sink | kitchen | 4 个餐具，pick up 和 place into sink 分别计分，最多 8 分 |
| Items in Drawer | kitchen | pick object、open drawer、put object、close drawer，最多 4 分 |
| Laundry in Basket | bedroom | pick clothing、place on/in basket、fully inside basket，最多 3 分 |
| Make the Bed | bedroom | blanket、两个 pillows、neatness，最多 5 分 |

### Language following setup

- 两个 unseen kitchen scenes。
- 每次给 5 个物体，指令要求移动其中一个。
- 目标物体故意放得比 distractors 更远，避免模型靠最近物体 shortcut。
- 指标：language following rate 和 task success rate。
- OOD 物体类别不在训练集中，例如 funnel、pill bottle、grill lighter、lighter、safety goggles。

### 关键 ablation 结论

1. **环境数量 scaling**：location 数量增加带来稳步提升，约 100 个家庭环境能接近 seen-test-home baseline。
2. **ME/CE 数据重要**：去掉跨 embodiment 数据会显著降低 household task performance。
3. **WD 影响语义泛化**：对 OOD object language following 尤其明显。
4. **HL 数据重要**：即使 runtime 不显式生成高层子任务，只要训练中包含 HL prediction，也能显著提升。
5. **VI 小但关键**：VI 只占 high-level mobile manipulation examples 约 11%，但去掉后明显退化。
6. **GPT-4 不是免费高层策略**：zero-shot GPT-4 高层 policy 最差，说明 VLM 必须被机器人数据适配。

## 批判性思考

### 优点

1. **评估设定强**：真实新家庭是比常见 lab benchmark 更难也更有说服力的评估。
2. **recipe 清晰**：把 MM、ME、CE、HL、WD、VI 的作用拆开 ablate，结论比单纯“scale works”更有用。
3. **系统闭环完整**：从数据、模型、动作表示、机器人控制到真实评估都打通。
4. **对 VLA 方向有启发**：说明 high-level semantic prediction 不是外接 planner 的专利，可以作为 VLA 自身训练任务。

### 局限性

1. **复现门槛极高**：核心家庭移动机器人数据、硬件系统和真实家庭评估都不可轻易复现。
2. **实验样本仍有限**：真实家庭评估比一般 robotics paper 强，但相对开放世界复杂度仍然小。
3. **prompt 简单**：论文自己承认模型处理的指令复杂度有限，复杂偏好、跨房间任务和长期记忆没有解决。
4. **物理鲁棒性未根治**：陌生把手、难开的柜门、遮挡 spill、重复开关抽屉等仍会失败。
5. **安全层薄**：低层控制没有额外 collision detection 或 trajectory planning，真实部署还需要更强 safety envelope。

### 我的判断

[[Pi05]] 是一篇强系统论文，不是容易复现的算法论文。它的核心技术含量在训练配方和数据组织，而不在某个新 loss。最值得借鉴的是：真实机器人泛化需要把 semantic supervision、cross-embodiment transfer、Web visual knowledge 和 target-platform action data 组合起来，而不是只靠“多收同一台机器人的更多 demonstration”。
